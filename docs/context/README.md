# Mavebot Context Map

This folder is the durable context surface for mavebot's Slack/Discord remote
Codex runner. Use it to make remote channel work feel like a local Codex
Desktop session without dumping every old chat message into every prompt.

## Load Order

Remote worker prompts include context in this order:

1. Active Slack or Discord request.
2. Project `AGENTS.md`.
3. This context map.
4. Runtime/deploy snapshot generated by the worker.
5. Worker `summary.md` and `recent.md` from the shared server context.
6. `operating-memory.md`.
7. `slack-session.md`.
8. Extra focused context files, currently prioritized as:
   `remote-codex-session.md`, `code-map.md`, `clash-ui-guidance.md`.
9. Current source code and tests.

The active request wins over older memory. Source code and tests are the final
authority when docs are stale.

## Files

- `operating-memory.md`: app identity, server paths, deploy behavior, command
  registration rules, safety boundaries, and remote-worker infrastructure.
- `slack-session.md`: user preferences, UX decisions, current goals, and open
  work from the long-running channel session.
- `remote-codex-session.md`: behavior contract for making Slack `#bot` and
  Discord `#codex` act like persistent Codex sessions.
- `code-map.md`: source-file map and feature change recipes.
- `clash-ui-guidance.md`: Clash of Clans data, icon, and Discord UI guidance.

## Maintenance Rules

- Add durable facts only when future jobs will use them.
- Move repeated implementation rules into focused files instead of bloating
  `slack-session.md`.
- Delete or rewrite duplicated stale notes after preserving the durable fact in
  the right file.
- Do not store secrets, raw `.env` values, OAuth tokens, private keys, cookies,
  or one-off temporary debugging chatter here.
- When behavior changes, update the docs that explain that behavior in the same
  commit as the code.
